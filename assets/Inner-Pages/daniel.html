Introduction:

Why I Did This Sometimes you just want to push your limits. I challenged myself to build and deploy a MERN stack application in just 48 hours. Here's the idea, the plan, what worked, what didn’t, and the lessons I learned.


The Project Idea:

A simple task manager with user auth, CRUD features, and dark mode—small enough to build fast but real enough to feel useful.


Diving into the Stack:
  • MongoDB for the database
  • Express.js for the backend
  • React for the frontend
  • Node.js as the runtime
  • Tailwind CSS for rapid styling
  • Deployed using Render & MongoDB Atlas

The Timeline

Day 1:

Project setup and initial commits
Basic backend API built (users, tasks)
Frontend wireframes and routing
Set up auth using JWT


Day 2:

Frontend functionality (create/edit/delete tasks)
Connected backend with frontend
Added dark mode and responsive design
Final polish and deployment


Challenges Faced:

JWT token handling took longer than expected, Dealing with CORS during deployment, Styling issues on mobile.


Conclusion

I didn’t sleep much, but I learned a ton. Building under pressure forced me to make smart, fast decisions. And yes, I pushed to GitHub every few hours just in case 😎.
